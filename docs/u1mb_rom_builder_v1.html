U1MB Rom Builder Help
  U1MB Rom builder is a tool to create/edit the ROM file used in flashing the AMIC A29040BL Flash Memory chip in the Ultimate 1MB upgrade board. The chip is 512K in size, holds various ROM images used in the operation of the U1MB, and can be customized to fit individual needs.  The memory map of the chip is as follows:
            HEX Offset DEC Offset ROM Description Size (K)
0x0 0x3FFFF 0 262,143 SDX (Spartados X) 256
  0x40000 0x4FFFF 262,144 327,679 Reserved (1) 64
  0x50000 0x53FFF 327,680 344,063 U1MB Bios 16
  0x54000 0x57FFF 344,064 360,447 Reserved 16
  0x58000 0x5FFFF 360,448 393,215 Reserved (2) 32
  0x60000 0x61FFF 393,216 401,407 BASIC 8
  0x62000 0x63FFF 401,408 409,599 Cartridge 1 8
  0x64000 0x65FFF 409,600 417,791 Cartridge 2 8
  0x66000 0x67FFF 417,792 425,983 Cartridge 3 8
  0x68000 0x69FFF 425,984 434,175 XEGS Game 1 (3) 8
  0x6A000 0x6BFFF 434,176 442,367 XEGS Game 2 (3) 8
  0x6C000 0x6DFFF 442,368 450,559 XEGS Game 3 (3) 8
  0x6E000 0x6FFFF 450,560 458,751 XEGS Game 4 (3) 8

  0x74000 0x77FFF 475,136 491,519 OS 2 16

(1) This ROM slot is defined as being part of a vacant SDX area in the official U1MB documentation. GUI is one of the possible uses for this vacant ROM space.
 
 (2) This ROM slot is defined as reserved in the official U1MB documentation.  PBI is one of the possible uses for this vacant ROM space.
 
(3) P3 jumper on the U1MB board must be ON for XEGS mode and OS 4 slot must be loaded with XEGS OS Rom.
 

How to use U1MB Rom Builder

You can use the ROM builder to create a new U1MB ROM or modify an existing one.

To create a new U1MB Rom file  simply start the program. U1MB Rom Builder will start with a base template ROM already loaded. or if you are already in the program click on the menu item (or button) File/New U1MB to start with a new base ROM. The template used is based on the default U1MB Rom that was loaded into the device at the time of the shipment.

A new U1MB Rom will be indicated with "untitled.rom" next to the main Window title on the titlebar. You can now customize the base Rom by loading your choice of individual Rom files by clicking on the ROM NAME and then clicking on the Load Rom button at the bottom of the screen.

Note that when you load a U1MB Bios Rom, the program will ask you whether you want to override the descriptions that are currently on the screen with the descriptions from the Bios Rom (Bios Rom is where all descriptions are stored, see the Technical Note at the end of this help page for more information). If you answer yes, all descriptions for BASIC, Game and OS Rom slots will be overriden by the descriptions from the Bios Rom file, otherwise the current  descriptions will be preserved and saved when you save the U1MB Rom file or the Bios Rom file (using Extract Rom button)

Also note that, there are 3 rom slots defined as Reserved. According to the currently available U1MB documentation these empty slots are reserved for future use, also all U1MB devices so far came with those slots empty. U1MB Rom Builder will allow you to rename those slots and save those definitions permanently should you choose to do so.

For example, you can rename the slot called RSVD1 (64K) to GUI (64K), or RSVD3 (32K) to PBI (32K) and assign a short (14 characters) description to them like "G.U.Interface", or "P.B.Interface". To do so, simply double-click on the RSVD1 (64K) item to enable it for input, enter a name of your choice and press ENTER, the description field for that slot will then be enabled for input, you can now enter your description. The information you entered will be saved and made permenant (until you choose to change it again). Only RSVD1, RSVD2 and RSVD3 slots can be modified this way. Also remember that these changes will have no effect on the actual contents of the U1MB Rom or on the U1MB Setup Menu, they are to customize the Rom Builder UI so that the reserved slots have meaningful names when they are actually used.

To modify an existing U1MB Rom file, start the program and load a valid U1MB Rom file by clicking on the File/Load U1MB Rom menu item or button. Once the U1MB Rom is loaded the Rom name will be shown next to the Main Window title, you can then select to update individual Rom files within it.

  • To update an individual Rom file,  select the Rom Name by clicking the appropriate name on the name column and click the LOAD Rom button under the Description column. You can also change the description of the Rom if you wish. Once you are finished updating individual Rom files click on the File/Save U1MB menu item or button to save the updated U1MB Rom.

  • To save an individual Rom file from within an U1MB Rom select a Rom Name from the ROM NAME column and click the Extract Rom button under the DESCRIPTION column.

To make an ATR file containing the U1MB Rom and the flasher software click on the File/Make ATR menu item or button . The ATR file is used to flash the Rom of U1MB using your Atari computer.

Technical note on ROM description strings: Individual Rom Descriptions and Description Headers are stored in the U1MB BIOS Rom. For example, the line for the BASIC slot on Ultimate 1MB Setup screen has two parts:

Description Header   Description
BASIC Slot is:         BASIC
BASIC Slot is:   CAR1
BASIC Slot is:   CAR2
BASIC Slot is:   CAR3

Above lines (each having a total of 32 bytes = 18 chrs Header + 14 chrs Description) are all stored in the U1MB BIOS Rom, note that the Description Header for each slot is the same, so when the user presses the right or left arrow key in the Ultimate Setup screen, the whole screen line (all 32 characters, not just the Description) change, but since you have the same Description Header for each Basic slot you only see a change in the Description itself (last 14 Characters).

U1MB Rom builder will only allow you to change the Description part of the string. So the maximum number of characters that can be used is 14. Spaces count as part of the description and are stored in the U1MB Bios Rom file. Those are the characters which actually describe the ROM. Some Rom Generating programs may allow you to change the Description Header as well as the Description. In my opinion it's best to keep the description headers untouched to ensure a consistant presentation on the U1MB setup screen.
